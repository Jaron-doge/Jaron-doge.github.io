B-树

  • 每d代合并：m = 2^d路， m - 1个关键码

  • 充分利用外存对批量访问的高效支持，将此特点转化为优点，每下降一层，都以超级节点为单位，读入一组关键码

  • m的大小视磁盘的数据块大小而定KB/4 = 200 ~ 300

  • 树高h = 外部节点深度

  • 内部节点的分支数n+1也不能太少

    • 树根：2 <= n+1 <= m
    • 其余： 「m / 2 ⌉ <= n + 1 <= m

  • 也称为(「m / 2 ⌉, m)-树

红黑树

  • 支持对历史版本的访问
  • 任何一次动态操作引发的结构变化量不超过O(1)
  • 红黑树与4阶B数相同

定义规则

  • 树根：黑色
  • 外部节点：均为黑色
  • 其余节点：若为红，则只能有黑孩子
  • 外部节点到根：黑节点数目相等
